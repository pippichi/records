## 中断和异常

![image-20260427224009716](操作系统.assets/image-20260427224009716.png)

中断是操作系统内核夺回CPU使用权的唯一途径

没有中断应用程序就会一直执行，也就没有并发这个概念了，因此中断很重要，可以说没有中断就没有操作系统

![image-20260427225423956](操作系统.assets/image-20260427225423956.png)



中断的类型

![image-20260428111903722](操作系统.assets/image-20260428111903722.png)



内中断例子：

CPU执行非法指令或指令的参数非法等情况时，会引发内中断信号

![image-20260428114204347](操作系统.assets/image-20260428114204347.png)

应用程序请求操作系统内核服务，此时会执行一条特殊指令“陷入指令”，该指令会引发一个内部中断信号，CPU会转向该中断信号相应的处理程序

![image-20260428114913790](操作系统.assets/image-20260428114913790.png)

陷入指令不是特权指令

时钟每隔一个时间片会给CPU发送时钟中断信号，CPU接到外中断信号切换为内核态处理该中断信号对应的内核程序

打印机处理完后会给CPU发送中断信号

每一条指令执行结束时，CPU都会例行检查是否还有外中断信号

外核

![image-20260428203449909](操作系统.assets/image-20260428203449909.png)

一般内核给进程空间申请一片连续内存空间是虚拟的，实际上真实存的地方其实是离散的；同理给进程空间申请一片连续的文件存储空间（外存磁盘空间）也是这样。

外核做的事情就是真实的分配一段连续的内存空间、一段连续的文件存储空间。作用：当有随机频繁访问文件的情景时磁头不需要来回横跳，效率更高。

外核没有访问资源时需要先经过VA（虚拟地址）映射到PA（实际物理地址）的开销。

同时外核还需要负责不重复分配已经被分配的，并且只允许访问被分配的空间而不允许访问除此之外的空间。简言之就是保证资源使用安全。

## 虚拟机

传统计算机一个物理机器只能运行一个操作系统，当物理机器很牛的时候会极大的浪费物理机性能

![image-20260428220942909](操作系统.assets/image-20260428220942909.png)

因此有人发明了虚拟机

同义术语：虚拟机管理程序/虚拟机监控程序/Virtual Machine Monitor/Hypervisor

![image-20260428221550917](操作系统.assets/image-20260428221550917.png)

第一类虚拟机，直接运行在硬件上

![image-20260428221748708](操作系统.assets/image-20260428221748708.png)

此类VMM有个问题，虚拟机管理程序运行在内核态，而上层操作系统并不知道自己运行在用户态所以它是有可能用到特权指令的，此时会被虚拟机管理程序截获替换为等价操作，让指令看起来是成功的

![image-20260428222449011](操作系统.assets/image-20260428222449011.png)



第二类虚拟机，运行在宿主机操作系统上，需要用虚拟机管理程序如VMWare等

![image-20260428222923328](操作系统.assets/image-20260428222923328.png)

这类虚拟机，想要分配资源只能请求宿主操作系统再通过虚拟机管理程序再分配，硬件资源管理者仍然是宿主操作系统

![image-20260428223540838](操作系统.assets/image-20260428223540838.png)

两类虚拟机管理程序对比

![image-20260428223603389](操作系统.assets/image-20260428223603389.png)

第二类VMM得到的虚拟内存需要先映射到宿主机虚拟内存再映射到实际物理内存，性能更差



支持虚拟化的CPU通常分更多指令等级

![image-20260428224746808](操作系统.assets/image-20260428224746808.png)

这样划分可以提高效率，比如：如果虚拟机内核空间运行在Ring1、虚拟机用户空间运行在Ring2那就可以直接执行下去，除非需要用Ring0级别的指令。

![image-20260428225035861](操作系统.assets/image-20260428225035861.png)



第二类VMM一部分运行在用户态、一部分运行在内核态，在内核态的部分是以虚拟机驱动的方式加载到宿主机操作系统中的

当虚拟机要执行特权指令时，虚拟机管理程序会代替它向底层宿主机操作系统发起指令，性能也会较差

![image-20260428225122646](操作系统.assets/image-20260428225122646.png)

### 线程的实现方式和多线程模型

![image-20260430095251597](操作系统.assets/image-20260430095251597.png)



用户级线程

![image-20260430095351537](操作系统.assets/image-20260430095351537.png)

![image-20260430095511593](操作系统.assets/image-20260430095511593.png)

![image-20260430095729361](操作系统.assets/image-20260430095729361.png)

1、用户级线程由线程库管理，并不是操作系统

2、用户态下就可以完成用户级线程的工作

3、操作系统只能看到进程看不到线程

4、优点：用户级线程在用户空间即可完成不需切换到核心态，开销小效率高；缺点：一个用户级线程被阻塞，整个进程都被阻塞并发度不高，多个线程无法在多核处理机上并行运行

![image-20260430100203044](操作系统.assets/image-20260430100203044.png)



内核级线程

![image-20260430100259211](操作系统.assets/image-20260430100259211.png)

1、内核级线程由操作系统内核管理

2、内核级线程的切换必须要在核心态下才能完成

3、操作系统会为每个内核级线程建立相应的TCB（Thread Control Block线程控制块）

4、优点：并发高，多线程可在多核处理机上并行执行；缺点：一个用户进程会占用多个内核级线程，线程切换需要切换到核心态，线程管理成本高开销大

![image-20260430100635822](操作系统.assets/image-20260430100635822.png)



上述两种线程都有各自优缺点，能否结合？

在支持内核级线程的机器下使用线程库将多个用户级线程映射到某个内核级线程。由此引出多个映射模型：

一对一模型

![image-20260430101524715](操作系统.assets/image-20260430101524715.png)

多对一模型

![image-20260430101704572](操作系统.assets/image-20260430101704572.png)

如果给进程分配多个内核级线程的话，在多核机器下也能并行处理，但考试中默认多对一只分配一个内核级线程



多对多模型★

优先级调度算法补充：

1、就绪队列未必只有一个，可以按照不同优先级来组织。另外也可以把优先级高的进程排在更靠近队头的位置；

3、系统进程优先级高于用户进程，前台进程优先级高于后台进程；

4、操作系统偏好IO繁忙型进程（与之对应的是CPU繁忙型进程）；

### 进程的内存映像

假设32位系统，进程虚拟地址空间为4G

高位1G为操作系统内核区，这块区域存储了PCB、页表、内核代码（比如进程调度）、内核数据结构等

程序被翻译后的机器指令、const修饰的常变量会放在“只读代码/数据”区域

## I/O控制器

![image-20260508100822520](操作系统.assets/image-20260508100822520.png)

IO设备的机械部件

![image-20260508100906907](操作系统.assets/image-20260508100906907.png)

IO设备的电子部件（IO控制器/设备控制器）

CPU无法直接控制IO设备的机械部件，因此IO设备还要有一个电子部件作为CPU和IO设备机械部件之间的中介实现CPU对设备的控制

![image-20260508102650462](操作系统.assets/image-20260508102650462.png)

1、IO控制器中的控制寄存器来存放命令和参数，如读多少字节写多少字节等

2、IO控制器中的状态寄存器用于记录IO设备当前的状态，如1表空闲，0表忙碌

3、IO控制器中的数据寄存器用于CPU与设备之间的数据传递

4、IO控制器中的地址识别：识别CPU要控制上述的哪种寄存器



IO控制器的组成

![image-20260508104630651](操作系统.assets/image-20260508104630651.png)

上图左边就是与CPU的接口，右边就是与设备的接口，中间是IO逻辑

一个IO控制器可以控制多个IO设备



CPU与控制器的通信：

1、CPU首先会通过控制线对IO控制器发送一个特殊指令，然后通过地址线告诉IO控制器要操纵哪个设备

2、如果CPU要输出数据的话会通过数据总线将数据放入数据寄存器中，之后IO逻辑会读取该数据

3、CPU发送指令时会有参数放在控制寄存器中，之后IO逻辑会读取该参数

4、为了实现对各个设备的管理，CPU会从状态寄存器中读取各个设备的状态，IO寄存器会往状态寄存器中写入各个设备的状态



设备与控制器的通信：

1、输入/输出：CPU ->/<- 数据寄存器 ->/<- IO逻辑 ->/<- 控制器与设备的接口 ->/<- 传输数据

2、设备反馈状态：传输状态 -> 控制器与设备的接口 -> IO逻辑 -> 状态寄存器

3、控制信息：IO逻辑根据CPU发出的命令和相应参数对对应的设备发出控制命令让设备执行具体工作



注意：上述各种寄存器都会有多个，每个寄存器都要有相应的地址，寄存器编制有两种方式：

1、内存映像IO；

2、寄存器独立编址；



内存映像IO vs 寄存器独立编址

![image-20260508111929851](操作系统.assets/image-20260508111929851.png)

内存映像IO会占用一些内存地址，如果内存地址是0-N-1，那么设备控制器的寄存器地址会继续往后编

优点：简化指令，可采取对内存进行操作的指令来对控制器进行操作



寄存器独立编址会单独编号

缺点：需要设计专门的指令来实现对控制器的操作，不仅要指明寄存器地址还要指明控制器编号

DMA控制器

主机-控制器接口：

DR（data register）、MAR（memory address register）、DC（data counter）、CR（command register）都是数据寄存器，用于CPU和控制器之间的通信，间接来控制IO设备。这些寄存器也有可能有多个

![image-20260508151451595](操作系统.assets/image-20260508151451595.png)

块设备-控制器接口：用于控制器和块设备的通信、控制

![image-20260508151529187](操作系统.assets/image-20260508151529187.png)

CPU可以在一开始就指明这次读入的数据是存放在内存什么位置以及写入磁盘什么位置（这些信息是存放在MAR里的），并且指明此次读入的数据量是多少（存在DC里）。接下来CPU就不需要干预了，DMA会完成整个任务，完成之后给CPU一个中断指令让CPU介入处理即可

注意DMA也不是一块一块读数据的，而是一个字一个字读，先存在DR然后再一次性处理达到块传输效果

![image-20260508154857813](操作系统.assets/image-20260508154857813.png)

注意每次读写必须是连续的块，并且这些块读入内存后也得是连续的，如果是离散的则CPU需要分多次请求DMA处理

![image-20260508160113150](操作系统.assets/image-20260508160113150.png)

为了解决离散导致的缺点，提出了通道控制方式

CPU、内存、通道通过总线连接在一起

CPU告诉通道任务清单（通道程序首地址），数据从哪里拿放哪里，之后通道根据任务清单完成所有任务后发起中断请求

由于通道只能识别单一指令且通道程序跟CPU共享内存，所以它也被称为弱鸡版CPU

注意：1个通道可以控制多个IO控制器，1个IO控制器可以控制多个IO设备

B的用户进程可以发起一个网络套接字socket（申请一片内核空间用于收发数据）

socket系统调用会给B返回一个描述符fd，然后给套接字bind本地端口6666，这样就可以等待被连接了

引导块

![image-20260510094132297](操作系统.assets/image-20260510094132297.png)

计算机开机时需要进行一系列初始化工作，这些工作通过执行初始化程序（自举程序）完成，自举程序放在ROM中由厂家集成在主板上不可修改

万一要更新自举程序怎么办？

现代计算机会改为在ROM中存放很小的“自举装入程序”，完整的自举程序在磁盘启动块（引导块/启动分区）上，位于磁盘固定位置

开机时计算机先运行“自举装入程序”找到引导块并将完整的自举程序读入内存完成初始化

u盘也是闪存技术Flash Memory

固态硬盘的结构

![image-20260510101044490](操作系统.assets/image-20260510101044490.png)

每个闪存芯片由若干块组成，块大小16KB-512KB

![image-20260510101209023](操作系统.assets/image-20260510101209023.png)

每个块又可以拆解多个页，页大小512B-4KB

![image-20260510101437613](操作系统.assets/image-20260510101437613.png)

系统对固态硬盘的读写是以页为单位的，磁盘读取是以块为单位的，因此：

固态硬盘的页 -> 磁盘的块/扇区

固态硬盘的块 -> 磁盘的磁道

![image-20260510102642921](操作系统.assets/image-20260510102642921.png)



固态硬盘以块为单位擦除，每个页只能写1次，读无限次，注意如果还想往同一个页写，则必须先擦除块

当一个块中某几页有数据时，怎么往某个有数据的页写？

答：先复制其他页到B块，然后在B块上写入新数据，再擦除原块。并且，闪存翻译层会将原先的映射舍弃，重新映射逻辑块号到新的物理块

![image-20260510103912164](操作系统.assets/image-20260510103912164.png)

所以对于固态硬盘来说，逻辑地址对应的物理地址可能会变，只不过闪存翻译层会将它修改正确



磨损均衡技术

固态硬盘的块写多了就会坏，因此需要将”擦除“平均分布在各个块上，提升寿命

1、动态磨损均衡：写的时候选较新的块；

2、静态磨损均衡：读多写少的内容迁移到老的块；
